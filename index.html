WHAT ARE .ONION DOMAINS?

It is a generic pseudo top-level domain that indicates an anonymous IP address accessible through the Tor network. Addresses are a combination of 16 alphanumeric characters systematically generated based on a public key when Tor is configured. This 16-character combination can be created with any letter of the alphabet and decimal digits starting with 2 and ending with 7, thus representing an 80-bit base32 number. They are part of the Deep Web. Although such addresses are not actually DNS, web search engines can access .onion sites by using a proxy and sending the request through servers in the Tor network. The goal of using this system is to make it difficult for both the distributor and the recipient of information to trace, either between themselves or by a third party. On the "surface" web, we can track or find websites through search engines. On the Deep Web, there are no search engines (only some facilitators); the user must know the website they are going to visit, often under a password. On the Deep Web, you can find the content of the traditional web, but also everything that doesn't appear on it. Goldshlag, Michael Reed, and Paul Syverson, applying the ideas of David Chaum to achieve networks that preserve privacy and anonymity (both of the message and the interlocutors) in a manner that is transparent to the communicating entities. Messages are repeatedly encrypted and sent through several nodes and networks (onion routers), and each node deletes the information from the previous node, preventing the source/destination information and the content of the message from being known. On this alternative network, page addresses are not in the usual format and are almost illegible and impossible to remember. They begin with HTTP (they are still web pages) and end with .ONION. To access the Deep Web, you must be connected to the Tor network and use a web browser like the Tor Project, which has versions for Windows, Linux, and Mac.
